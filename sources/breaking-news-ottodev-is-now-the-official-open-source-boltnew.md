# BREAKING NEWS: oTToDev is Now the OFFICIAL Open Source Bolt.new

Cole announces that oTToDev, the community project he and the automator "think tank" had been building as an open-source AI coding assistant, is entering a strategic partnership with [StackBlitz](../entities/organizations/stackblitz.md) to become the official open-source version of [bolt.new](../entities/tools/bolt-new.md), renamed [bolt.DIY](../entities/tools/bolt-diy.md). He sits down with StackBlitz CEO Eric Simons to lay out the shared vision: bolt.new remains the stable commercial product for easily building anything, while bolt.DIY is the fast-moving, experiment-friendly, run-it-yourself version that supports swapping in any LLM (including [local models](../concepts/local-ai.md)) and pushes cutting-edge features first. This is a defining case of [open-source AI coding](../concepts/open-source-ai-coding.md) driven by [community development](../concepts/community-driven-development.md).

A recurring theme is that bolt.DIY has become a real-world [coding benchmark](../concepts/coding-benchmarks.md) for frontier model labs. Because it exercises models against actual full-stack application generation rather than synthetic evals, labs now ask "can it run Bolt?" the way PC gamers once asked "can it run Crysis?" Cole notes he personally test-drives every new model in bolt.DIY, citing spinning up [Qwen 2.5 Coder 32B](../entities/tools/qwen.md) within hours of its release. A second pillar of the vision is education: teaching people how to actually work with [AI coding assistants](../concepts/ai-coding-assistant.md) as co-pilots rather than assuming the model is broken when they get bad results.

Cole then covers the transition logistics: the automator think tank stays the community home, the repo moves into the StackBlitz Labs GitHub org (keeping the same stars, forks, history, PRs, and issues), the community retains full creative freedom over roadmap and direction, and commercial bolt.new will deliberately diverge from bolt.DIY so each can pursue its own vision while still sharing ideas. He acknowledges the sting of losing the oTToDev name but frames bolt.DIY (do-it-yourself) as fitting the run-it-yourself, local-LLM, cutting-edge goals of the project.
