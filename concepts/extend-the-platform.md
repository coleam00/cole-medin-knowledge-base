# Extending a Platform via API + Marketplace

Extending a platform is a business strategy: instead of building a product from scratch, you add AI functionality on top of an established SaaS platform that already has users, billing, and data. The opportunity exists when a platform is genuinely developer-first. Cole's example is GoHighLevel: "go high level also has an incredible API so it is so easy to extend the platform like their API is so good that you would think that they are a developer first platform." [0:01:38] A great API is what makes the whole niche viable, it lowers the cost of building integrations to near zero.

The second ingredient is a **marketplace**, which supplies the plumbing every SaaS integration needs so you do not have to build it yourself. As Cole puts it, "these Marketplace apps make it really easy to handle authentication if you're building a SAS integrate with their API set up web hooks to handle events." [0:02:09] Authentication, [webhook](./webhook-driven-agent.md) event handling, and permissions come as part of the platform, so your work collapses to the actual AI value you are adding. The marketplace is also distribution: your extension is discoverable by the platform's existing customer base.

Why this is profitable rather than just convenient: you inherit an audience and a payment rail instead of acquiring them. The platform has already solved user onboarding, data, and monetization; you attach an [agent](./agent-as-api-endpoint.md) to the events that matter and charge for the outcome. It is one of the clearer [AI agent business niches](./ai-agent-business-niches.md) precisely because the hard, unglamorous parts are already built.
