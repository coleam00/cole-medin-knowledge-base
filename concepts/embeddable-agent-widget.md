# Embeddable Agent Widget

An embeddable agent widget is the simplest way to put an agent in front of real users: you drop a single snippet onto a page and the agent appears, no custom frontend required. As Cole describes it, "you also have the embed code right here a script tag if you want to have it as a chat widget or an iframe if you want it as a part of your website itself." [0:10:34] Because it is just a script tag or an iframe, it works on essentially any host: "you just click on this option right here and then you take the script tag that you can embed into really any website builder." [0:13:00]

The pattern gives you a few placement options for the same underlying agent. As Cole lays them out, "there are three different options you can have a floating chat where it's like that little widget in the bottom right that someone clicks on very common on websites or you can have it inline or full screen." [0:12:34] A floating bubble is the familiar customer-support pattern; inline embeds the agent inside existing page content; full-screen turns it into a dedicated page. The choice is presentation only, the agent behind all three is identical.

This is the deployment surface that makes agents commercially useful without building a full application. It even extends to voice, "this is where we can hook in the workflow that we just made with audio input and output into a voice spot that we can actually embed as a chat widget on our website or through an iframe as well." [0:09:08] For lead-gen and client work, an embeddable widget is often the entire deliverable: it is a [frontend-agnostic](./frontend-agnostic-agent.md) way to ship an [agent API](./agent-as-api-endpoint.md) onto a client's existing site.
