# Learn by Building

Learn by building is Cole's conviction that you actually understand AI engineering only by making things with it, not by passively consuming tutorials. "I'm a very hands-on learner. And honestly, I believe pretty firmly that everyone is," he says, because "I always learn a lot more when I actually do something... you want to learn by doing." The claim is not just pedagogical taste; it is that building is what produces real understanding and control over what your systems do.

The strongest version of the idea is the build-it-yourself principle. Rather than adopting a powerful open-source assistant wholesale, "the balance we should strike here is we should learn from these powerful open-source assistants that are out in the wild, but build something ourself so we truly understand and control the solution." The payoff is ownership: "I'm a proponent of building your own solution because then you truly understand and control what your agent is doing and you can customize it to the exact level of control that you need." This is also why so many of Cole's projects are subpar-tool replacements, "most of these platforms are straight up just not good, so you might as well just build the agent yourself."

Forking is the on-ramp. Open source lets you learn by taking something real and changing it, "it is also open source, so going to the GitHub right here you can download all of the code... what makes it even better is you can extend this application because you have access to all the source code," and "it's a really cool lesson for you to see how you can change applications when you fork them." Cole models this by forking the local AI starter kit and adding his own services to it. The other engine of learning-by-building is docs and examples: give the coding assistant "a lot of documentation and examples," including "a link to a GitHub repo that has an already existing implementation... using this example kind of as documentation," and learn to read SDK documentation directly "instead of relying on YouTube videos to teach you how to do any basic thing." Follow a structured path of real builds and "you're going to be comfortable creating any AI application you could possibly dream of."
