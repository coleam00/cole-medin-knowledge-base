# Open-Source & Local AI Coding

Open-source AI coding tools are community-built, self-hostable coding assistants that you can run yourself, most importantly against free local models. The motive is cost and control: with local LLMs "you never have to pay a dime or hit those nasty rate limits ever again" [0:00:20]. That directly answers the pain of hosted tools, where users "slowly use more and more credits as they're making these changes and then they hit their limit and they can't continue." Running the fork yourself means "you can take our initial project from lovable or bolt.new and then tweak it to your hearts content completely for free."

## How it works

The flagship example in these videos is the Bolt.new fork that grew into oTToDev, the community effort "we are building to be the best open source AI coding assistant." Being open source is itself the mechanism: "you can run yourself and use to work with all these local llms with all these insanely cool features that commercial bolt.new won't have because we can just move so fast as an open source repository." Development is [community-driven](./community-driven-development.md), "so many of you are making awesome additions... and our core team of maintainers has really really stepped up," which lets the fork ship capabilities (local-model support, local project loading) faster than the closed product. The result is a tool "that is accessible to everyone."
