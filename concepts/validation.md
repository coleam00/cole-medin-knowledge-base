# Validation

Validation is building explicit checks into an agentic workflow so the AI verifies its own output against real success criteria before it moves on. It is the "V" in Cole's core build loop: "validation is also a very important part of what I like to call the PIV loop... validation is all about giving your coding assistant the ability to check its own work after it has finished an implementation" ([AI Fixes My Code Better than Me Now](../sources/ai-fixes-my-code-better-than-me-now-heres-how.md), 0:00:24). Without it, autonomy is just vibe coding: "AI generated code is still your responsibility. So you need to validate. I'm not going to be a proponent of Vibe coding" ([This One Command Makes Coding Agents Find All Their Mistakes](../sources/this-one-command-makes-coding-agents-find-all-their-mistakes-use-it-now.md), 0:01:02).

Cole organizes validation as a pyramid. The agent owns the cheap, deterministic base and humans own the layers that matter most: "I'll be covering what I call the AI validation pyramid. This is a framework that allows us to define the validation requirements before we even write a single line of code" ([Claude Code's Agent Teams Are Insane](../sources/claude-codes-agent-teams-are-insane-multiple-ai-agents-coding-together-in-real-time.md), 0:08:47). "we have our agent handle the foundation, all the easy things like the type checking and linting and our initial round of testing. And then we as the humans control the layers that matter most" (0:09:05). Defining those requirements up front, in the PRP, is essential: "You always want to describe to the AI coding assistant in the PRP. What do you need to do to actually validate that your code is good?" ([Build ANY AI Agent with this Context Engineering Blueprint](../sources/build-any-ai-agent-with-this-context-engineering-blueprint.md), 0:19:02).

Mechanically, validation runs as an iterate-until-green loop with a bounded retry budget before escalating to a human. Stripe's coding agents follow exactly this: "after the coding agent writes some set of code, we are going to run the linting and the type checking, the unit testing. If there are any failures, then we loop back to the agent and have it fix these things and retry" ([Stripe's Coding Agents](../sources/stripes-coding-agents-ship-1300-prs-every-week-heres-how-they-do-it.md), 0:03:00), capped so "we iterate here a maximum of two times before we escalate to a human because we don't want infinite loops with unit test errors" (0:12:18). A separate scoring agent can gate on a threshold: "it's going to give a score of 1 to 10 for each one of these criteria... as long as it meets the threshold or exceeds, then the implementator did a good job" ([Coding Agent Reliability Explodes When They Argue](../sources/coding-agent-reliability-explodes-when-they-argue-new-adversarial-dev-technique.md), 0:11:30). The goal is to shrink the human bottleneck: "by the time control passes back to us for our human validation, there's less that needs to be corrected... We want to reduce us being the bottleneck" ([Principled Agentic Engineer](../sources/full-guide-to-becoming-a-principled-agentic-engineer-build-anything-with-ai.md), 0:52:08).

Validation is not only about code. You validate the plan before executing it ("you want to validate the PRP before you execute it. Make sure that it's aligned with what you want to create and that it's not overengineering" - [Archon Beta Launch](../sources/archon-beta-launch-livestream-what-you-missed.md), 0:36:30), validate an idea before spending credits generating a video ("we want to sort of like validate the idea for the video before we generate the video" - [Video Generation System with Archon](../sources/i-turned-claude-code-into-a-complete-video-generation-system-with-archon.md), 0:06:12), and validate visual output by rendering and screenshotting it ("it renders the PNG, views the image, finds any imperfections, and then it'll just make direct edits to the JSON file" - [Build BEAUTIFUL Diagrams with Claude Code](../sources/build-beautiful-diagrams-with-claude-code-full-workflow.md), 0:07:45). It can even be fully deterministic: an MCP that checks a knowledge graph "doesn't use an LLM for analysis at all. It's all just deterministic code" ([AI Code That Fixes Itself](../sources/ai-code-that-fixes-itself-an-mcp-you-can-try-now.md), 0:04:10). Google's agentic-engineering framing captures the endgame: "if you really want the most reliable code possible that's where you want systematic verification at every stage" ([Google Masterclass on Agentic Engineering](../sources/google-just-dropped-a-masterclass-on-agentic-engineering-its-so-good.md), 0:06:23). And it isn't going anywhere: "you can review the AI generated code because validation is also not going away anytime soon" ([Is Software Engineering Finally Dead?](../sources/is-software-engineering-finally-dead.md), 0:15:52).

## Part of

- [The PIV Loop](./the-piv-loop.md) - validation is the "V" stage of Prompt-Implement-Validate.
- [AI Coding Harness](./ai-coding-harness.md) - validation gates and checkpoints are core harness components.
- [Phases of Work](./phases-of-work.md) - validation is the final phase before control returns to the human.

## Builds on

- [PRD-First Development](./prd-first-development.md) - success criteria are defined in the spec before any code.
- [Reduce Assumptions](./reduce-assumptions.md) - explicit checks replace blind trust in the model's claims.

## Contrasts with

- [Vibe Coding](./vibe-coding.md) - shipping unread, unverified AI output; validation is its deliberate opposite.

- [End-to-End Validation](./end-to-end-validation.md) - the highest, most rigorous layer of the pyramid.
- [Code Review](./code-review.md) - reading the diff for bugs and quality as a validation gate.
- [Separate Reviewer](./separate-reviewer.md) - an unbiased grader that improves validation reliability.
- [Test-Driven Development](./test-driven-development.md) - tests written first give validation an objective target.
- [Human in the Loop](./human-in-the-loop.md) - the human owns the top layers of the validation pyramid.
- [Subagents Pattern](./subagents-pattern.md) - validation runs in its own sub-agent context.
- [MCP](./mcp.md) - MCP servers give agents tools to verify deployed apps and knowledge graphs.
- [RAG](./rag.md) - source validation lets a RAG agent verify its answers against approved chunks.
