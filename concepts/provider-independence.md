# Provider Independence

Provider independence is architecting an agent so the model, framework, or client underneath can be swapped without a rewrite. Cole states the imperative directly: "You don't want to be stuck with just one model or provider for your entire AI coding workflow. It's just not the best way to do things these days," and the personal design goal is "I want to be very LLM agnostic." The urgency is the pace of the field: "I see so many people get super married to using specific tools... that is so dangerous because of how fast the AI landscape is evolving," since "you never know when that tool is going to become irrelevant." The durable investment is not the tool but the surrounding system: "Building a system that can scale that can work with any tools that you have."

At the model layer, the mechanism is almost trivial: the OpenAI chat-completions API is a de facto standard, so "the only thing we have to do to connect to Ollama instead of OpenAI is change this base URL." Frameworks like [Pydantic AI](../entities/tools/pydantic-ai.md) reduce a swap to "just changing this line or just changing my environment variable... A single line change. I can swap to any model I want like Gemini or DeepSeek or OpenAI." This is why Cole favors low-level abstractions (see [Abstraction Distraction](./abstraction-distraction.md) and the [SDK vs. Framework decision](./sdk-vs-framework-decision.md)): "they are a lowlevel abstraction... they don't try to do too much for you so you have to write more code but then you get that full control and customizability," including "being able to store the message history yourself." The recurring subtext across the Bolt.diy fork saga is the same fix applied to a coding tool that shipped locked to one model.

At the harness layer, independence means the [AI layer](./the-ai-layer.md) and workflows outlive the agent running them. Because commands and [skills](./skills.md) are just prompt files, "I could very easily take this repository, swap out Claude Code for Codex, and then still use the exact same harness because it is just those artifact files," and a slash command "can just be a prompt for any other AI coding assistant besides Claude Code." The result is deliberate mixing: "All of the top engineers right now, they're not relying on a single model or even a single harness... because we want to lean on the different strengths of the different AI coding assistants," e.g. Claude as implementer and Codex as evaluator, or Opus planning while Gemini designs. A strong harness even lifts a cheaper model: driving everything with Pi on Kimi still yields "really good results because of the harness that I built here that elevates the model." Emerging standards generalize the pattern: the [Agent Client Protocol](./agent-client-protocol.md) lets you "move between these code editors... try different ones seamlessly," and AG-UI decouples front ends so "you're not just limited to using OpenAI."
