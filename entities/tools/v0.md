# v0

v0 is Vercel's AI front-end generator: describe an interface in natural language and it writes [Next.js](./nextjs.md) and [shadcn/ui](./shadcn-ui.md) components, rendering code and live preview side by side. Cole's framing is that "this platform makes it so so easy to use AI to build robust front-end components or even build entire front ends" ([0:00:00]). One detail he insists on is which v0 you use: "keep in mind that there's a different version of vzer if you don't go to the chat page if you go to just vz. deev that one I don't like as much" ([0:01:45]). The chat surface is the one worth using.

What makes it more than a mockup tool is the handoff. v0 emits an npx install command that pulls the generated component straight into a real local Next.js project, so the output is code you own rather than a hosted preview. From there Cole iterates prompt by prompt, adding webhook calls into an [n8n](./n8n.md) agent, markdown rendering, a dark theme, then [Supabase](./supabase.md) auth. Where v0 stumbles (quote-style import bugs, a default-versus-named export error) he pastes the error plus the full component into [Claude 3.5 Sonnet](./claude-3-5-sonnet.md), which diagnoses it exactly. That pairing, generator for structure and a strong model for debugging, is the actual workflow.

He is equally clear about when to skip it. Wrapping an agent in a chat UI does not always justify a React project: "you don't have to create a full react or nextjs project like you would with vzero" ([0:00:48]), and a [Streamlit](./streamlit.md) app written by Claude gets there faster. He leaves the tradeoff open rather than settling it: "there is a time and place to use v0 and also decode a front end yourself but that is a discussion for another video" ([0:01:03]).

## Realizes

- [Rapid Prototyping](../../concepts/rapid-prototyping.md) - Get a real front end standing in minutes so your time goes to the agent behind it.
- [Full-Stack AI Application](../../concepts/full-stack-ai-application.md) - The UI half of an agent app with auth, memory, and a webhook-backed backend.
- [Iterative Prompting](../../concepts/iterative-prompting.md) - Prompt for a minimal baseline, then add one change per turn.
- [Everyday AI Coding Use Cases](../../concepts/ai-coding-use-cases.md) - Offloading the front-end work a backend-focused builder would rather not hand-write.
- [Browser-Based AI Coding Tools](../../concepts/browser-based-ai-coding.md) - In-browser generation that gets you started fast but needs a real editor for refinement.

## Contrasts with

- [Streamlit](./streamlit.md) - The faster, plainer alternative Cole picks when a chat wrapper does not warrant a React app.
- [Lovable](./lovable.md) - AI app builder covering the same generate-a-front-end territory.
- [bolt.new](./bolt-new.md) - StackBlitz's platform that generates entire full-stack apps in a web container rather than components you export.
- [Builder.io](./builder-io.md) - Figma-to-code product Cole cites as the canonical case where training a custom model beat general GPT, then used GPT to refine the generated code.

## Works with

- [Next.js](./nextjs.md) - The framework v0 targets, and the local project the npx install command drops components into.
- [shadcn/ui](./shadcn-ui.md) - The component library the generated UI is built from.
- [Claude 3.5 Sonnet](./claude-3-5-sonnet.md) - The debugging partner for the import and export errors v0 leaves behind.
- [Supabase](./supabase.md) - Auth for the generated app, with the signed-in user ID reused as the agent's session ID.
- [n8n](./n8n.md) - The agent backend the generated chat UI calls over a webhook.
