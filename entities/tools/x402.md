# Coinbase x402

x402 is Coinbase's payment protocol for agents that bill other agents. Cole defines it as the missing settlement layer under the agent economy: "Coinbase recently released a protocol for this called X42. It's a way for us to build AI agents that we expose over the internet, but we require payment" [0:11:50] whenever someone else interacts with them. Strip away the crypto and it is a metering standard: your agent becomes a paid endpoint, and the caller settles automatically instead of signing up, entering a card, and waiting on an invoice.

He raises it as the concrete mechanism behind [machines paying machines](../../concepts/machines-paying-machines.md), one of his 2026 predictions and, by his own admission, the newest idea in that video. The argument for stablecoins is functional rather than ideological: these networks need "a currency where it's really easy to do micro payments quickly and globally," something stable enough to price against, so "a stable coin like USDC is the perfect option for this" [0:12:44]. Card rails do not do sub-cent, cross-border, machine-speed transfers. That is the gap x402 fills.

The sharper claim is that x402 may rescue [agent-to-agent protocols](../../concepts/agent-protocols.md). Cole diagnoses [A2A](../../concepts/a2a-protocol.md) as stalled by a chicken-and-egg problem: an A2A-compatible agent has nothing to talk to until a network already exists, so the value proposition collapses at launch. Payment breaks that deadlock by supplying a reason to join before the network is large, since "we can make money with our agents assuming it exposes capabilities that other people want to leverage through their AI agents." Monetization becomes the bootstrap incentive that pure interoperability never had.

Practically, adopting x402 means treating your agent as a product: an [API endpoint](../../concepts/agent-as-api-endpoint.md) with per-call pricing, deployed to a real host, [authorizing](../../concepts/agent-authorization.md) and isolating callers the way any [multi-tenant](../../concepts/multi-tenant-agents.md) service must. It turns the [business-niche](../../concepts/ai-agent-business-niches.md) question from "which humans will subscribe" into "which agents will call this, and what is one call worth."

## Works with

- [Agent2Agent (A2A) Protocol](../../concepts/a2a-protocol.md) - Google's open standard for how independently-built agents discover and talk to each other over HTTP via agent cards and tasks, the agent-to-agent counterpart to MCP.
- [Agent Protocols](../../concepts/agent-protocols.md) - Emerging standards (A2A, agent cards, discovery) that let agents interoperate and be called across clients.
- [Remote MCP Server](../../concepts/remote-mcp-server.md) - An MCP server hosted in the cloud rather than run locally, making tool access scalable, shareable, and deployable behind a production URL.
- [Multi-Tenant Agents](../../concepts/multi-tenant-agents.md) - Building a single agent that serves many users, each with personalized tools, memory, and securely authorized access to their own accounts.
